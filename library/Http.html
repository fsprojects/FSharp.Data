F# Data: HTTP Utilities

The .NET library provides powerful API for creating and sending HTTP web requests. There is a simple WebClient type (see MSDN) and a more flexible HttpWebRequest type (see MSDN). However, these two types are quite difficult to use if you want to quickly run a simple HTTP request and specify parameters such as method, HTTP POST data or additional headers.

The F# Data Library provides a simple Http type with two overloaded methods: Request and AsyncRequest that can be used to create a simple request and perform it synchronously or asynchronously.

To use the type, we first need to reference the library using #r (in an F# interactive) or add reference to a project. The type is located in FSharp.Net namespace:

1: #r "../../bin/FSharp.Data.dll"
2: open FSharp.Net

Sending simple requests

To send a simple HTTP (GET) request that downloads a specified web page, you can use Http.Request and Http.AsyncRequest with just a single parameter:

1: // Download the content of a web site
2: Http.Request("http://tomasp.net")
3: 
4: // Download web site asynchronously
5: async { let! html = Http.AsyncRequest("http://tomasp.net")
6:         printfn "%d" html.Length }
7: |> Async.Start

In the rest of the documentation, we focus at the Request method, because the use of AsyncRequest is exactly the same.

Query parameters and headers

You can specify query parameters either by constructing a URL that includes the parameters (e.g. http://...?test=foo&more=bar) or you can pass them using the optional parameter query. The following example also explicitly specifies the GET method, but it will be set automatically for you if you omit it:

1: Http.Request("http://httpbin.org/get", query=["test", "foo"], meth="GET")

Additional headers are specified similarly - using an optional parameter headers. The collection can contain custom headers, but also standard headers such as the Accept header (which has to be set explicitly when using HttpWebRequest).

The following example uses The Movie Database API to search for the word "batman". To run the sample, you'll need to register and provide your API key:

1: // API key for http://www.themoviedb.org
2: let apiKey = "<please register to get a key>"
3: 
4: // Run the HTTP web request
5: Http.Request
6:   ( "http://api.themoviedb.org/3/search/movie",
7:     query   = [ "api_key", apiKey; "query", "batman" ],
8:     headers = [ "accept", "application/json" ])

Sending request data

If you want to create a POST request with HTTP POST data, you can specify the additional data in a string using the body parameter, or you can specify a set of name-value pairs using the bodyValues parameter. If you specify body data, you do not need to set the meth parameter - it will be set to GET automatically. The following example uses the httpbin.org service which returns the request details:

1: Http.Request("http://httpbin.org/post", bodyValues=["test", "foo"])

By default, the Content-Type header is set to application/x-www-form-urlencoded, but you can change this behaviour by adding content-type to the list of headers using the optional argument headers:

1: Http.Request
2:   ( "http://httpbin.org/post", 
3:     headers = ["content-type", "application/json"],
4:     body = """ {"test": 42} """)

Maintaing cookies across requests

If you want to maintain cookies between requests, you can specify the cookieContainer parameter. The following example will request the MSDN documentation for the HttpRequest class. It will return the code snippets in C# and not F#:

1: // Build URL with documentation for a given class
2: let msdnUrl className = 
3:   let root = "http://msdn.microsoft.com"
4:   sprintf "%s/en-gb/library/%s.aspx" root className
5: 
6: // Get the page and search for F# code
7: let docInCSharp = Http.Request(msdnUrl "system.web.httprequest")
8: docInCSharp.Contains "<a>F#</a>"

If we now go to another MSDN page and click on a F# code sample, and then go back to the HttpRequest class documentation, while maintaining the same cookieContainer, we will be presented with the F# code snippets:

 1: open System.Net
 2: let cc = CookieContainer()
 3: 
 4: // Send a request to switch the language
 5: Http.Request
 6:   ( msdnUrl "system.datetime", 
 7:     query = ["cs-save-lang", "1"; "cs-lang","fsharp"], 
 8:     cookieContainer = cc) |> ignore
 9: 
10: // Request the documentation again & search for F#
11: let docInFSharp = 
12:   Http.Request
13:     ( msdnUrl "system.web.httprequest", 
14:       cookieContainer = cc )
15: docInFSharp.Contains "<a>F#</a>"

If you want to see more information about the response, including the response headers, the returned cookies, and the response url (which might be different to the url you passed when there are redirects), you can use the RequestDetailed method:

1: let response = Http.RequestDetailed(msdnUrl "system.web.httprequest")
2: 
3: // Examine information about the response
4: response.Cookies
5: response.ResponseUrl

Requesting binary data

The Request method will always return the response as a string, but if you use the RequestDetailed method, it will return a HttpResponseBody.Text or a HttpResponseBody.Binary depending on the response content-type header:

1: let logoUrl = "https://raw.github.com/fsharp/FSharp.Data/master/misc/logo.png"
2: match Http.RequestDetailed(logoUrl).Body with
3: | HttpResponseBody.Text text -> 
4:     printfn "Got text content: %s" text
5: | HttpResponseBody.Binary bytes -> 
6:     printfn "Got %d bytes of binary content" bytes.Length
